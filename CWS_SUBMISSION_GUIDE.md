# 크롬 웹스토어 배포용 등록 정보 안내 (Chrome Web Store Submission Guide)

크롬 개발자 대시보드에서 신규 항목 등록 시 필요한 모든 텍스트 정보와 개인정보 보호 관행 탭 입력 내용을 정리해 두었습니다. 그대로 복사하여 붙여넣으시면 됩니다.

---

## 1. 스토어 등록 정보 (Store Listing)

### 1-1. 단일 목적 설명 (Single Purpose Description)
> **[입력 위치: 스토어 등록 정보 ➔ 요약]**
```text
국가유산 보고서(PDF)를 다운로드할 때 웹 페이지의 정보(발행기관, 발행연도, 보고서제목, 파일 순서 번호 등)를 자동으로 추출하여 일관되고 가독성 높은 파일명으로 자동 변환하여 저장합니다.
```

### 1-2. 상세 설명 (Detailed Description)
> **[입력 위치: 스토어 등록 정보 ➔ 상세 설명]**
```text
국가유산포털(heritage.go.kr), 문화유산지식포털(portal.nrich.go.kr), 정부24(e-minwon.go.kr) 등 유산 조사 보고서 제공 사이트에서 복잡한 파일명을 참고문헌식 표준 파일명([발행기관], [발행연도], [보고서제목] [순서번호].pdf)으로 자동 변환해 줍니다. 또한 현재 페이지의 보고서 정보를 기반으로 『조사기관명, 발행년도, 『보고서명』』 형태의 학술 인용 양식을 원클릭으로 클립보드에 복사할 수 있어 인공학 및 역사학 연구 효율을 높여줍니다.
```

### 1-3. 카테고리 (Category)
*   **카테고리**: `생산성` (Productivity) 또는 `개발자 도구` (Developer Tools)

### 1-4. 언어 (Language)
*   **언어**: `한국어` (Korean)

---

## 2. 개인정보 보호 관행 (Privacy Practices)

### 2-1. 권한 요청 이유 (Permission Justification)
> **[입력 위치: 개인정보 보호 관행 ➔ 권한 선언 설명]**

#### ① storage 권한 사용 이유
```text
사용자가 지정한 파일명 배열 규칙(템플릿)과 동기화 설정 상태를 브라우저 기기 간에 저장하고 유지하기 위해 사용합니다.
```

#### ② downloads 권한 사용 이유
```text
다운로드 이벤트 발생 시 추출한 학술 보고서 메타데이터를 기반으로 파일명을 정리하여 알맞은 규칙으로 로컬에 저장하기 위해 사용합니다.
```

#### ③ 호스트 권한 (Host Permissions) 사용 이유
*   대상 사이트: `https://www.heritage.go.kr/*`, `https://www.cha.go.kr/*`, `https://www.khs.go.kr/*`, `https://www.e-minwon.go.kr/*`, `https://portal.nrich.go.kr/*`, `http://116.67.83.213/*`
```text
국가유산포털, 지식이음, 국립문화유산연구원 등의 학술 보고서 보기 및 다운로드 웹 페이지 DOM에 콘텐츠 스크립트를 삽입하여 보고서명, 발행기관 등의 서지 메타데이터를 추출하기 위해 사용합니다.
```

### 2-2. 원격 코드 사용 여부 및 이유 (Remote Code Justification)
> **[입력 위치: 개인정보 보호 관행 ➔ 원격 코드 사용 여부]**
*   **선택**: `아니요, 원격 코드를 사용하지 않습니다.`
```text
해당 확장 프로그램은 어떠한 원격 코드도 실행하거나 로드하지 않으며, 모든 코드는 로컬 패키지 내부에 포함되어 보안 및 신뢰성을 보장합니다.
```

### 2-3. 데이터 사용 및 정책 준수 인증 (Data Usage Certification)
> **[입력 위치: 개인정보 보호 관행 ➔ 정책 준수 서명]**
*   "개발자 프로그램 정책을 준수하는 방식으로 데이터를 사용한다는 사실을 인증합니다" 항목에 **체크**

## 3. 스토어 이미지 자산 (Store Assets)

웹스토어 등록에 필요한 이미지들이 프로젝트 루트의 `store_assets/` 폴더에 준비되어 있습니다:

1.  **프로모션 타일 이미지 (1280x800)**: `store_assets/promo_tile.png`
2.  **스크린샷 1 (파일명 설정 화면)**: `store_assets/popup_rename.png`
3.  **스크린샷 2 (보고서 인용 화면)**: `store_assets/popup_citation.png`

---

## 4. 계정 및 설정 절차 안내

1.  **이메일 인증**:
    *   크롬 개발자 대시보드 ➔ **설정** 페이지로 이동하여 게시자의 연락처 이메일 주소를 입력하고, 수신된 이메일의 인증 링크를 클릭하여 인증절차를 완료합니다.
2.  **게시물 업로드**:
    *   `archreport` 폴더 전체를 `.zip` 파일로 압축하여 대시보드에 업로드합니다.
